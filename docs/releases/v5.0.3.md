# WeFlow 5.0.3

## 更新内容

- 修复部分会话导出时出现 `status -3 / cursor state failed` 的问题；原生导出不可用时自动切换兼容导出。
- 导出任务复用主进程的 WCDB 连接，避免多个原生连接互相重置游标状态。
- 游标读取失败时停止写入，避免生成内容为空或不完整的导出文件。
- 消息数据库索引改为每个账号只扫描一次，不再在每个页面统计请求前重复扫描。
- 为会话、联系人、群成员、年度报告与各类统计加入内存缓存和并发请求合并；年度年份直接复用已持久化统计结果。
- 会话列表不再等待公众号未读扫描，耗时工作延后到后台执行。
- 超大群成员首次加载超时后继续在后台完成，返回页面时直接使用已完成的缓存。
